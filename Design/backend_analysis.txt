### рџ”ђ Auth System ([auth.controller.js](cci:7://file:///d:/ViaPool/Backend/src/controllers/auth.controller.js:0:0-0:0) + [auth.middleware.js](cci:7://file:///d:/ViaPool/Backend/src/middlewares/auth.middleware.js:0:0-0:0))

- **Dual-token strategy**: short-lived **access token** (cookies + `Authorization: Bearer`) + long-lived **refresh token**
- Refresh tokens are stored as an **array** in MongoDB (supports multi-device login)
- `verifyJWT` middleware attaches `req.user` to protected routes
- `setupDriverProfile` uses a **MongoDB transaction** to atomically update the user role + create a `Vehicle` document

### рџљ— Ride System ([ride.controller.js](cci:7://file:///d:/ViaPool/Backend/src/controllers/ride.controller.js:0:0-0:0))

- Only users with `role: "driver"` can `createRide`
- **Geospatial search**: `searchRides` uses MongoDB's `$geoNear` aggregation pipeline вЂ” finds rides within **50km radius** of the passenger's coordinates, sorted by departure time, paginated
- The `Ride` model stores `from` and [to](cci:1://file:///d:/ViaPool/Backend/src/utils/ApiError.js:1:4-19:5) with GeoJSON [Point](cci:1://file:///d:/ViaPool/Backend/src/controllers/booking.controller.js:9:0-20:2) coordinates and **2dsphere indexes** for fast spatial queries
- `availableSeats = totalSeats - 1` (driver's own seat is reserved at creation)

### рџ“‹ Booking System ([booking.controller.js](cci:7://file:///d:/ViaPool/Backend/src/controllers/booking.controller.js:0:0-0:0))

- **Atomic seat reservation** via `findOneAndUpdate` with `{ availableSeats: { $gte: seatsBooked } }` вЂ” prevents race conditions when multiple passengers book simultaneously
- `cancelBooking` uses a **MongoDB transaction** to atomically mark booking cancelled AND restore `availableSeats` on the ride
- Drivers can view all passengers for their rides via `getRidePassengers`

### вљЎ Real-time (Socket.IO вЂ” [socket/index.js](cci:7://file:///d:/ViaPool/Backend/src/socket/index.js:0:0-0:0))

Socket connections are **JWT-authenticated** (reads token from `auth` handshake, `Authorization` header, or cookie). Three key real-time features:

| Event | Description |
|---|---|
| `join-user-room` | User joins their personal room `user_<userId>` for notifications |
| `join-ride-room` | Joins `ride_<rideId>` to receive location updates |
| `location-update` (driver sends) | Broadcasts `driver-location` to all passengers in the ride room |
| `send-message` | Saves message to MongoDB, emits `receive-message` to receiver's personal room |
